# 2026-02-28

## 变动 恢复页面级垂直滚动
### 原因 页面右侧滚动条缺失，鼠标滚轮不生效
### 影响 允许 body 垂直滚动；页面统一承担纵向滚动，横向滚动保持在主容器中
### 验证 npm run lint 通过；在水平/垂直布局下手动验证滚轮与滚动条表现

## 变动 保留横向滚动并允许页面纵向增长
### 原因 h-screen 固定视口高度 + 容器 overflow-y-hidden 导致纵向滚动被抑制
### 影响 外层容器改为 min-h-screen；水平布局移除 overflow-y-hidden 让纵向溢出由页面承载
### 验证 在水平布局下横向滚动条可见；页面右侧纵向滚动条可见，滚轮滚动正常

## 变动 新增 When Free 状态列与下拉项
### 原因 需求希望在状态下拉中选择 When Free，并在状态视图中看到对应列
### 影响 状态种子增加 When Free；老库自动补齐该状态；状态下拉与颜色映射同步扩展

## 变动 新增日期视图 (Date View) 与想法记录窗体
### 原因 提供按日期组织任务的视角，以及快速记录带时间戳的想法
### 影响 
- 数据层：Task 新增 createdAt 字段；老数据自动回填当前时间
- UI：新增 Date 视图模式（按日期分列）；新增 IdeaModal 用于快速添加想法（支持选日期）
### 验证
- 切换到 Date 视图，列按日期显示
- 点击 New 按钮可记录想法并选择日期，保存后归档至对应日期列
- 现有 Status/Context 视图功能不受影响

## 变动 优化日期视图列交互
### 原因 日期列非持久化实体，原有的重命名、移动、拖拽等操作无效且误导用户；折叠功能此前未生效
### 影响 
- Date View：隐藏列菜单（移动/重命名）与拖拽手柄；标题输入框设为只读
- Date View：启用列折叠功能（基于本地状态）
### 验证
- 在 Date View 下点击折叠按钮可正常收起/展开列
- 在 Date View 下不再显示菜单按钮与拖拽图标
- 标题不可编辑

## 变动 支持日期列持久化与全功能交互
### 原因 用户期望日期视图列也能像普通列一样被管理（拖拽排序、重命名、持久化折叠状态）
### 影响 
- 数据层：新增 `date_columns` 表；自动同步任务日期到该表
- UI：恢复 Date View 下的列菜单、拖拽手柄和标题编辑功能
- 交互：支持日期列的排序、重命名和状态持久化
### 验证
- 在 Date View 下可以拖拽排序列
- 可以重命名日期列（如将 "2026-03-01" 改为 "Deadline"）
- 折叠状态刷新页面后保持不变
